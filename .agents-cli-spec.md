# Agent Spec

## Overview
Bodhi is a Unified Personal "Second Brain" Assistant graph workflow agent. It provides a single universal input portal (text) where a user logs their day conversationally. The agent uses a cloud-hosted LLM to route and parse the unstructured input into distinct database objects (expenses, time logs, habits, nutrition, sleep, somatic symptoms, journaling).

## Example Use Cases
**Input:** "Spent $15 on paneer tikka, took 1 hour, checking off my study habit, felt great today."
**Output:** Concurrently updates financial (expense), nutritional (food items), time (duration, activity), habit, and journal databases.

## Tools Required
- `nutrition_router`: Converts Indian food descriptions into macro/calorie estimates.
- `expense_router`: Categorizes expenses and infers merchants.
- `time_auditor`: Parses natural language durations.
- `habit_tracker`: Validates and checks off habits.
- `sleep_tracker`: Logs sleep patterns and computes duration.
- `somatic_logger`: Captures physical symptoms and severity.
- `journal_logger`: Generates insights, mood scores, and tags.
*(These will be simulated via mock functions/database handlers in the prototype phase)*

## Constraints & Safety Rules
- Must use ADK 2.0 Graph Workflow API.
- Must execute quickly.
- Must maintain a strict multi-entity JSON response schema.

## Success Criteria
- Given a complex daily log, the agent successfully routes the data to the correct distinct domain schemas (Expense, Nutrition, Time, Habit, Journal, Sleep, Somatic) via a fan-out graph architecture.
