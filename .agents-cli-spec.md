# Agent Spec

## Overview
Sage is a Unified Personal "Second Brain" Assistant graph workflow agent. It provides a single universal input portal (text) where a user logs their day conversationally. A router `LlmAgent` parses the unstructured input against a strict Pydantic schema (`SageAgentOutput`) in one call, and the graph fans out to 7 domain-processing nodes (nutrition, expenses, time, habits, sleep, somatic, journal) that are joined into a single combined summary.

## Example Use Cases
**Input:** "Spent $15 on paneer tikka, took 1 hour, checking off my study habit, felt great today."
**Output:** Concurrently updates the nutrition, expenses, time, habits, and journal domains from one router call.

## Graph Nodes
- `router_agent` (`LlmAgent`): Extracts structured data across all 7 domains in a single call, constrained by the `SageAgentOutput` schema.
- `process_nutrition` / `process_expense` / `process_time` / `process_habit` / `process_sleep` / `process_somatic` / `process_journal` (`FunctionNode`): Format each domain's parsed data into a human-readable summary line.
- `merge` (`JoinNode`) → `combine_results` (`FunctionNode`): Combine all non-empty domain summaries into a single response event.

Deeper analysis (mood scoring, cross-domain correlations, weekly insight synthesis) is handled separately by async specialist `LlmAgent`s in `app/specialists.py`: `journal_enricher`, `sleep_analyzer`, `somatic_correlator`, `expense_analyzer`, `time_analyzer`, `insight_synthesizer`. These run in the background via the Node.js backend's job queue, not inline in the router graph.

## Constraints & Safety Rules
- Must use ADK 2.0 Graph Workflow API.
- Must execute quickly.
- Must maintain a strict multi-entity JSON response schema.

## Success Criteria
- Given a complex daily log, the agent successfully routes the data to the correct distinct domain schemas (nutrition, expenses, time_logs, habits, sleep, somatic_logs, journal) via a fan-out/fan-in graph architecture.
