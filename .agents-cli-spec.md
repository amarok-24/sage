# Agent Spec

## Overview
Sage is a Unified Personal "Second Brain" Assistant graph workflow agent. It provides a single universal input portal (text) where a user logs their day conversationally. The agent uses a cloud-hosted LLM to route and parse the unstructured input into distinct database objects (resources, time logs, rhythms, replenish, rest, reactions symptoms, reflections).

## Example Use Cases
**Input:** "Spent $15 on paneer tikka, took 1 hour, checking off my study rhythms, felt great today."
**Output:** Concurrently updates financial (resources), replenishal (food items), time (duration, activity), rhythms, and reflections databases.

## Tools Required
- `replenish_router`: Converts Indian food descriptions into macro/calorie estimates.
- `expense_router`: Categorizes resources and infers merchants.
- `time_auditor`: Parses natural language durations.
- `habit_tracker`: Validates and checks off rhythms.
- `sleep_tracker`: Logs rest patterns and computes duration.
- `somatic_logger`: Captures physical symptoms and severity.
- `journal_logger`: Generates insights, mood scores, and tags.
*(These will be simulated via mock functions/database handlers in the prototype phase)*

## Constraints & Safety Rules
- Must use ADK 2.0 Graph Workflow API.
- Must execute quickly.
- Must maintain a strict multi-entity JSON response schema.

## Success Criteria
- Given a complex daily log, the agent successfully routes the data to the correct distinct domain schemas (Resources, Replenish, Time, Rhythms, Reflections, Rest, Reactions) via a fan-out graph architecture.
